## Research Findings

This report is an implementation-ready specification for a text->first-order-logic->Prolog neuro-symbolic pipeline whose admission boundary is a label-free knockoff+ FDR gate, plus the fair operational comparison that maps every comparator into a common (head, Re-DocRED-relation, tail) triple space at matched recall.

BLOCK A - EXTRACTION. LINC uses the LLM purely as a semantic parser into NLTK/Prover9 FOL syntax (quantifiers 'all x.'/'exists x.', connectives & | -> -, predicates Pred(arg1,arg2)), prompting with a fixed instruction block, few-shot 'TEXT:\tNL\nFOL:\tformula' pairs, and a test block delimited by <PREMISES>/<CONCLUSION>/<EVALUATE> tags with stop word '</EVALUATE>' [1,2,3]. Logic-LM is closer to our Prolog target: for ProofWriter/PrOntoQA it emits a four-section logic program (Predicates with ':::' glosses, Facts P(a1..an), Rules using the '>>>' implication operator, Query) defined by the grammar facts/rules(F1..Fm -> Fm+1..Fn)/queries [4,5]. The closest CLUTRR->logic precedent is LLM-ASP, which semantic-parses CLUTRR/StepGame/bAbI into reusable ASP (Clingo) modules [18]. RECOMMENDED on-disk forms: a FACT is a ground atom fact(rel_child,e_alice,e_bob) with a controlled functor and entity-id args plus a side etype/2 type table; a BRIDGE is a Prolog rule (composition bridge rel_grandmother(X,Z):-rel_mother(X,Y),rel_mother(Y,Z) for CLUTRR, or a surface->schema alignment rule for Re-DocRED). Bridges are first-class admission candidates with their own decoy/score/gate. Over-generation: instruct the extractor to emit every plausible (explicit AND lightly-inferred) atomic relation, one JSONL line each with a provenance span, at temperature 0.7 with n=3 samples unioned, capped at 20 candidates/document (the hypothesis budget assumption); copy-pasteable over-generation prompts for both Re-DocRED and CLUTRR configs and the candidate-record JSON schema (id, doc_id, kind, functor, args, arg_types, surface_span, raw_text, head/tail_surface, relation_phrase, source_system, explicit, raw_confidence) are given, with worked CLUTRR and Re-DocRED examples [3,4].

BLOCK B - PROLOG EXEC + TRACE-GRAPHS. RECOMMENDED bridge is janus-swi (pip install janus-swi; needs SWI-Prolog >= 9.2; official, bundled, ~5x lower overhead than pyswip, dict-based injection-safe param passing) with pyswip (pip install -U pyswip; ctypes; oldest interface) as fallback and a 'swipl' subprocess over a generated .pl as the safety net; the decision is made at runtime via a version probe + import smoke-test [6,7,8,9]. Exact APIs are documented: janus.consult/query_once/query/apply_once and runtime assertz; pyswip Prolog().assertz/query (a generator of binding dicts)/consult/register_foreign [7,9]. Proof trees are captured with a vanilla meta-interpreter (solve(true,true); solve((A,B),...):-solve(A),solve(B); solve(A,(A:-Proof)):-clause(A,B),solve(B,Proof)) extended so every base leaf resolves against a side table returning a certificate term cert(Wi,T,q,Span,FDPhat,r), i.e. solve(A,leaf(A,Cert)):-admitted_fact(A,Cert) and a node(...) form recording which bridge fired [10]. The trace-graph (nodes=subgoals/derived facts, edges=labelled rule applications, leaves carrying provenance + decoy_certificate (W_i,T,q) + entrapment_certificate (FDP-hat,r)) is serialized to JSON for machine audit and to Graphviz DOT/SVG (pip install graphviz + apt graphviz) for human audit; a 2-hop CLUTRR DOT example is provided.

BLOCK C - ARGUMENT TYPING. Coarse vocabulary {PER,LOC,ORG,TIME,NUM,MISC} aligned to DocRED NER. RECOMMENDED offline default is WordNet via NLTK (nltk.download('wordnet')): head noun -> wn.synsets(word,pos=NOUN) -> .hypernym_paths() -> presence of anchor synsets person.n.01->PER, location.n.01/region.n.03->LOC, organization.n.01/social_group.n.01->ORG, time_period.n.01->TIME, number.n.02/measure.n.02->NUM, else MISC; exact code given [20]. Re-DocRED's gold mention 'type' field is reused directly when present [11]; CLUTRR args all type to PER. ConceptNet 5 REST (api.conceptnet.io/c/en/<term>, IsA edges) and DBpedia SPARQL rdf:type dbo:* are OPTIONAL network enrichers, off by default [21,22].

BLOCK D - THE FIXED FAIR-MAPPING CORE. Re-DocRED documents are JSON objects with title, sents (list of token lists), vertexSet (list of entity clusters, each a list of mentions {name,sent_id,pos=[start,end),type}), and labels [{h,t,r,evidence}] where h/t index vertexSet and r is a Wikidata P-code [11]; files are data/{train,dev,test}_revised.json [13]. rel_info.json (P-code->name) ships with the original DocRED download (not in the public git repos); ~20 representative rows are tabulated (e.g. P17 country, P131 located in the administrative territorial entity, P27 country of citizenship, P569 date of birth, P175 performer, P40 child, P22 father, P25 mother, P26 spouse, P3373 sibling) [13,26]. The official metric (evaluation.py) counts a prediction correct iff its (title, r, h_idx, t_idx) tuple matches a gold tuple built as std[(title,r,h_idx,t_idx)]=set(evidence); precision=correct/submitted, recall=correct/tot_relations, Ign-F1 excludes train facts [12,13]. Claim-decomposition: one deterministic prompt turns CoT/RAG/conformal free-text into atomic (subject,relation_phrase,object) JSONL triples grounded in the document; for the neuro-symbolic system admitted facts/bridges are already triples (trivial pass-through). Relation-alignment is a deterministic HYBRID applied identically to every system AND to gold surface forms (a quality probe): (1) sentence-transformers all-MiniLM-L6-v2 cosine shortlist of the top-8 P-codes; (2) a fixed temperature-0 LLM classifier that picks one P-code or NO_RELATION; tie-break by cosine; fallback embedding-only argmax with a 0.45 floor [25]. Entity-linking normalizes and matches claim head/tail against every vertexSet 'name' in three tiers (exact -> alias/substring -> embedding floor 0.6); unmatched -> drop (counts against recall uniformly). Each system exposes ONE scalar per triple (neuro-symbolic W_i; plain raw confidence; CoT/RAG verbalized confidence; conformal frequency+gpt score); thresholds are swept to a common recall point where atomic-fact precision and the hallucinated-conclusion rate (fraction of multi-hop derived conclusions whose (h,r,t) is not entailed by gold) are compared, with CIs from a DOCUMENT-block bootstrap (resample whole documents, B=1000). Two honest controls: entity-linking misses and aligner error hit all systems symmetrically, so relative comparison plus matched-recall is preserved.

BLOCK E - BASELINES (all sharing one base model). RAG: sentence chunking + BM25 (rank_bm25 BM25Okapi) top-5 retrieval + a JSONL triple-generation prompt; CoT: think-then-emit-JSONL-triples. The PLAIN zero-label confidence-threshold gate (PRIMARY foil) scores each candidate in isolation with the SAME verbalized-[0,1] confidence prompt the decoy path uses, then admits above a swept threshold with no decoy/competition - isolating the gating mechanism at equal budget. The Mohri-Hashimoto conformal back-off is the LABELED reference: Algorithm 1 computes per-example scores r=inf{t: for all j>=t, Y* in E(F_j(x))}, sets q-hat as the ceil((n+1)(1-alpha))/n quantile, and outputs L-bar=F_{q-hat}; the sub-claim back-off keeps only sub-claims scoring >= t [14]. Its reference implementation (sayless.py/calibrate_thresh.py) is directly reusable: a verbatim BREAKDOWN_PROMPT for sub-claim decomposition with a gpt-score, a frequency score from 5 extra samples scored +1 supports/-1 contradicts/0 unrelated, a combined frequency+gpt score, and a ceil-quantile calibration that REQUIRES a human-annotated calibration set - exactly the labeled requirement the label-free decoy wedge removes [14,15]. Retained sub-claims map into the shared triple space via Block D. RECOMMENDED shared base model: a sub-$0.30/M OpenRouter model with prompt caching (DeepSeek-V3 implicit, or Gemini 2.x Flash implicit) [23]; structure scoring prompts as shared-document-prefix-first/per-candidate-suffix-last to exploit caching.

The decoy-gating core uses the knockoff statistic W_i=(Z_i v Z~_i)[2*1(Z_i>Z~_i)-1] and the knockoff+ data-dependent threshold T=min{t in {|W_i|}: (1+#{W_i<=-t})/max(1,#{W_i>=t}) <= q}, controlling FDR<=q with no labels [19,24]. The full report (research_report.md) additionally contains the unified ASCII pipeline diagram, a recommended Python library table (janus-swi/pyswip, nltk, sentence-transformers, rank_bm25, graphviz, numpy/scipy, openai client), CLUTRR/Re-DocRED data-format cheat-sheets, and 10 resolved decisions with fallbacks.
